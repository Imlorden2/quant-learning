# 1.1.3 市场微观结构

## 1. 订单簿（Order Book）

### 1.1 基本概念
- **定义**：记录所有买卖委托的电子账本
- **结构**：买盘（Bid）+ 卖盘（Ask）的价格-数量层级
- **实时更新**：每笔委托都会实时改变订单簿状态

### 1.2 订单簿结构示例
```
卖盘（Ask）          买盘（Bid）
价格    数量         价格    数量
12.05   1000        12.02   2000  ← 最优买价
12.04   1500        12.01   1800
12.03   2000  ← 最优卖价  12.00   3000
```

### 1.3 关键信息
- **最优买价（Best Bid）**：买盘最高价格
- **最优卖价（Best Ask）**：卖盘最低价格
- **深度（Depth）**：各价位的挂单数量
- **价格层级**：不同价位的分布

## 2. 买卖价差（Bid-Ask Spread）

### 2.1 定义与计算
```
买卖价差 = 最优卖价 - 最优买价
相对价差 = 买卖价差 / 中间价 × 100%
中间价 = (最优买价 + 最优卖价) / 2
```

### 2.2 价差示例
```
最优买价：12.02元
最优卖价：12.03元
买卖价差：0.01元
中间价：12.025元
相对价差：0.083%
```

### 2.3 影响因素
- **流动性**：流动性好的股票价差小
- **波动性**：波动大的股票价差大
- **交易量**：成交活跃的股票价差小
- **股价水平**：高价股绝对价差大，低价股相对价差大

### 2.4 不同股票价差对比
| 股票类型 | 绝对价差 | 相对价差 | 流动性 |
|----------|----------|----------|--------|
| 大盘蓝筹 | 0.01元 | 0.05% | 极好 |
| 中盘股票 | 0.01-0.02元 | 0.1% | 良好 |
| 小盘股票 | 0.02-0.05元 | 0.2%-0.5% | 一般 |
| ST股票 | 0.01-0.03元 | 0.3%-1% | 较差 |

## 3. 市场深度（Market Depth）

### 3.1 深度指标
- **一档深度**：最优买卖价位的挂单量
- **五档深度**：前五个价位的总挂单量
- **十档深度**：前十个价位的总挂单量

### 3.2 深度分析
```
深度比率 = 买盘深度 / (买盘深度 + 卖盘深度)
> 0.5：买盘力量较强
< 0.5：卖盘压力较大
= 0.5：买卖均衡
```

### 3.3 深度与交易的关系

- **浅深度**：大单容易造成价格冲击
- **深深度**：可以容纳较大交易量
- **不对称深度**：暗示价格变动方向

## 4. 流动性（Liquidity）

### 4.1 流动性定义
在不显著影响价格的情况下，快速买卖一定数量证券的能力。

### 4.2 流动性衡量指标

#### 4.2.1 成交量指标
```
日均成交量 = 近N日成交量总和 / N
换手率 = 成交量 / 流通股本 × 100%
```

#### 4.2.2 时间指标
```
零成交时间比例 = 无成交的分钟数 / 总交易分钟数
平均成交间隔 = 交易时间 / 成交笔数
```

#### 4.2.3 价格冲击指标
```
价格冲击 = |成交价格 - 中间价| / 中间价
市场冲击成本 = 冲击程度 × 交易金额
```

### 4.3 流动性分级
| 流动性等级 | 日均成交额 | 换手率 | 特征 |
|------------|------------|--------|------|
| 极佳 | >10亿 | >2% | 大盘蓝筹 |
| 良好 | 1-10亿 | 1%-2% | 主流中盘股 |
| 一般 | 0.1-1亿 | 0.5%-1% | 小盘股 |
| 较差 | <0.1亿 | <0.5% | 冷门股 |

## 5. 交易成本分析

### 5.1 交易成本构成
```
总交易成本 = 显性成本 + 隐性成本

显性成本 = 佣金 + 印花税 + 过户费
隐性成本 = 价差成本 + 冲击成本 + 时机成本
```

### 5.2 具体计算

#### 5.2.1 价差成本
```
价差成本 = 买卖价差 / 2
相对价差成本 = 价差成本 / 股价 × 100%
```

#### 5.2.2 冲击成本
```
临时冲击 = 实际成交价 - 决策时中间价
永久冲击 = 成交后价格 - 决策时中间价
```

### 5.3 成本优化策略
- **分批交易**：大单拆分减少冲击
- **时间分散**：避开开盘收盘等高波动时段
- **算法交易**：TWAP、VWAP等执行算法
- **暗池交易**：避免信息泄露

## 6. 量化交易中的应用

### 6.1 流动性筛选
```python
def liquidity_filter(stock_data):
    # 流动性筛选条件
    conditions = [
        stock_data['avg_volume_20d'] > 1e6,  # 日均成交量>100万股
        stock_data['avg_amount_20d'] > 1e8,  # 日均成交额>1亿
        stock_data['turnover_rate'] > 0.5,   # 换手率>0.5%
        stock_data['zero_volume_ratio'] < 0.1  # 零成交时间<10%
    ]
    return stock_data[all(conditions)]
```

### 6.2 交易成本预算
```python
def estimate_trading_cost(price, volume, spread, volatility):
    # 显性成本
    commission = price * volume * 0.0003  # 万三佣金
    stamp_tax = price * volume * 0.001 if is_sell else 0
    transfer_fee = price * volume * 0.00002
    
    # 隐性成本
    spread_cost = spread / 2 * volume
    impact_cost = volatility * (volume ** 0.6) * price * 0.1
    
    total_cost = commission + stamp_tax + transfer_fee + spread_cost + impact_cost
    return total_cost
```

### 6.3 最优执行策略
- **小额交易**：直接市价单
- **中额交易**：限价单+时间分散
- **大额交易**：算法交易+暗池
- **紧急交易**：接受较高成本快速执行

## 7. 微观结构异常

### 7.1 常见异常
- **价差异常放大**：流动性枯竭信号
- **深度突然消失**：大单冲击或撤单
- **单向倾斜**：买卖力量失衡
- **频繁撤单**：可能的操纵行为

### 7.2 异常处理
- **暂停交易**：等待市场恢复正常
- **调整策略**：降低交易量或改变执行方式
- **寻找替代**：转向流动性更好的品种
